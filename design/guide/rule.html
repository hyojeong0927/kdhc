Rule

목적

웹퍼블리싱 가이드는 웹사이트를 개발, 운영, 유지보수하는 과정에서 코드 품질, 디자인 일관성, 사용자 경험(UX)을 최적화하기 위한 표준을 제시합니다.
이 가이드는 팀 내 의사소통을 효율화하고 프로젝트 진행 시 일관된 결과물을 생성하는 데 목적이 있습니다.

기본 원칙

  1. 일관성: 모든 페이지에서 디자인 및 코드가 통일성을 유지해야 합니다.
  2. 접근성: WCAG(Web Content Accessibility Guidelines) 2.1 AA 기준을 준수합니다.
  3. 성능 최적화: 로딩 속도를 최소화하고 효율적인 코드 작성 원칙을 따릅니다.
  4. 반응형 디자인: 다양한 기기 및 해상도에서 일관된 사용자 경험을 제공합니다.
  5. 유지보수 용이성: 코드 가독성과 재사용성을 높이는 구조로 작성합니다.

HTML 코딩 가이드

  1. 문서 구조
    • HTML5 표준 준수
    • 시맨틱 태그 활용 (header, main, section, article, footer, address 등)
  2. 네이밍 규칙
    • 케이스: 소문자 사용
    • 구분자: 하이픈(`-`) 사용
    • 명명법: 명확하고 간결하게

      예: `main-header`, `footer-links`, `product-card`

  3. 접근성
    • 이미지: 대체 텍스트(`alt`) 제공
    • 폼 요소: 라벨(`label`)과 연결
    • 키보드 네비게이션 테스트(웹접근성 마크업 프로젝트일 경우에 한해)
  4. 기타 규칙
    • 중복 ID 금지
    • 필수 메타 태그 설정: `meta charset="UTF-8"`, `meta name="viewport" content="width=device-width, initial-scale=1.0"`
    • 주석 예시

CSS 코딩 가이드

  1. 구조 및 작성 규칙
    • CSS 파일 분리: 기본, 레이아웃, 컴포넌트, 유틸리티로 파일 분리
    • 코드 순서

      (1) 레이아웃 스타일

      (2) 컴포넌트 스타일

      (3) 유틸리티 스타일

  2. 네이밍 방식
    • BEM(Block-Element-Modifier) 규칙 준수
    • 블록: `.card`
    • 엘리먼트: `.card__title`
    • 수정자: `.card--highlighted`
  3. 단위 및 속성 우선순위
    • 단위: 상대 단위 (`em`, `rem`, `%`) 사용 권장
    • 속성 순서: 박스 모델(Box Model) 기준으로 정렬
  4. Sass Compiler
    • vscode 확장 프로그램 Live Sass Compiler 설치
    • .vscode > settings.json : 컴파일 경로 설정
    • vscode 우측 하단 Watch Sass를 클릭하여 실시간 컴파일

JavaScript 코딩 가이드

  1. 코드 스타일
    • ES6 표준 사용
    • 변수 선언: `const`, `let`만 사용
    • 함수: 화살표 함수 사용 권장
  2. 파일 구조
    • 주요 기능별 모듈 분리
    • `/js`
    • `/modules`: 재사용 가능한 컴포넌트
    • `/utils`: 공통 유틸리티 함수
  3. 이벤트 처리
    • 이벤트 위임 사용
    • 익명 함수 대신 명명된 함수 사용

테스트 및 QA

  1. 브라우저 호환성 테스트
    • 주요 브라우저: Chrome, Firefox, Edge, Safari
  2. 접근성 검사
    • Lighthouse, Axe, WAVE 도구 활용
  3. 반응형 테스트
    • 해상도별 테스트: 모바일(375px), 태블릿(768px), 데스크톱(1920px)

버전 관리

  1. Git 브랜치 전략
    • 기능 개발: `feature/[기능명]`
    • 버그 수정: `bugfix/[버그명]`
    • 릴리즈: `release/[버전명]`
  2. 커밋 메시지 템플릿
    • `feat`: 새로운 기능 추가
    • `fix`: 버그 수정
    • `docs`: 문서 추가/수정
    • `style`: 코드 스타일 변경 (CSS 등)
    • `refactor`: 코드 리팩토링

업데이트 및 유지보수

  1. 주요 업데이트 내역 기록 (CHANGELOG 관리)
  2. 업데이트 시 모든 팀원에게 공지
  3. 변경 사항은 QA 검토 후 배포
